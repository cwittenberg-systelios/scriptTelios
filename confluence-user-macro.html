## ## scriptTelios – Confluence User Macro ## ════════════════════════════════════════════════════ ## ## INSTALLATION (einmalig, als Confluence-Admin): ## 1. Administration → User Macros → Create User Macro ## 2. Macro name: scriptTelios ## 3. Macro title: scriptTelios ## 4. Body: No macro body ## 5. Template: Kompletten Inhalt dieser Datei einfügen → Save ## ## VERWENDUNG: ## Seite bearbeiten → Insert → Other Macros → "scriptTelios" einfügen ## ## BACKEND-URL KONFIGURIEREN (empfohlen): ## Beim Einfügen des Macros den Parameter "Backend URL" setzen. ## Damit müssen Therapeuten die URL nicht selbst eingeben. ## Beispiel: https://abc123-8000.proxy.runpod.net ## ## Bei Pod-Neustart: Macro bearbeiten → neue URL eintragen → Speichern. ## Alle Nutzer der Seite bekommen automatisch die neue URL. ## ## Alternativ: URL-Feld leer lassen → Therapeuten konfigurieren ## die URL beim ersten Aufruf selbst (wird im Browser gespeichert). ## ## ════════════════════════════════════════════════════════════════════════ ## @param backendUrl:title=Backend URL|type=string|required=false|default=|desc=URL des scriptTelios-Backends (z.B. https://abc123-8000.proxy.runpod.net). Wenn gesetzt, müssen Therapeuten nichts konfigurieren. ## @param authSecret:title=Auth Secret|type=string|required=false|default=|desc=Shared-Secret für die HMAC-Authentifizierung mit dem Backend (siehe AUTH-MIGRATION.md). Muss identisch mit CONFLUENCE_SHARED_SECRET im Backend sein. ## Container – äusserer Wrapper (vom Macro befüllt) + innerer leerer Mount-Punkt
